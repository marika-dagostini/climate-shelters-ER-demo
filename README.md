# Climate Shelters in Emilia-Romagna – Demo Web Application

This repository contains a demonstration version of an interactive web-based visual analytics system designed to support municipal decision-making for the planning of climate shelters (“climate oases”) in the Emilia-Romagna region, Italy.

The system enables the integration and exploration of environmental, socio-demographic, and infrastructural data through a multi-layer interactive map.

---

## 🌍 Demo Scope

The demo is **limited to the municipality of Bologna** and is intended to illustrate the core functionalities of the system.

**Important note**  
Due to privacy and data protection constraints:
- Health-related data are **not included**
- Some layers are **disabled**

Despite these limitations, the demo preserves the main visualisation and interaction features of the full system.

---

## 🧠 Project Context

This work was developed within the **Climate Oases Project** promoted by the Emilia-Romagna Regional Authority, which aims to support municipalities in identifying priority areas for climate shelter implementation.

The system is designed as a **decision-support tool**, allowing the exploration of spatial vulnerability through the integration of heterogeneous datasets.

---

## 🗺️ Features

- Multi-layer interactive map
- Dynamic layer selection (on/off)
- Transparency-based compositing
- Integration of heterogeneous datasets at different spatial scales
- Population-weighted visualization
- Infrastructure overlays (e.g., public buildings, healthcare facilities)
- Contextual side panels describing datasets and indicators

---

## ⚙️ Technical Workflow

The system was developed through the following pipeline:

1. Data processing and integration using **Stata, Excel, and QGIS**
2. Spatial harmonisation and preparation of datasets
3. Export to **GeoPackage** format
4. Generation of the web application using the **O.GIS Open Source WebGIS**
5. Deployment as an interactive web-based visualisation system

---

## 🚀 How to Use

1. Open the web application
2. Activate layers using the control panel
3. Adjust transparency to explore relationships between datasets
4. Combine multiple layers to identify areas where vulnerability factors overlap
